# MiTo — Design Context

## Design Context

### Users
**Primary audience**: Hackathon judges and observers at Buildathon Turin 2026. The interface must impress on first glance — it is a demo product, not a daily-driver tool. Secondary context: a personal finance power user who aggregates Revolut, PayPal, UniCredit, Intesa SanPaolo, and Splitwise into a single AI-surfaced view.

The job to be done: give the user (and AI) an instant, authoritative snapshot of their financial life across all accounts and social debts.

### Brand Personality
**Sleek, confident, futuristic.** MiTo should feel like a premium fintech product that exists in 2030 — not Mint or MoneyDashboard, but something that makes judges think "this could be a real product." Ambitious, technically sophisticated, zero fuss.

### Aesthetic Direction
**Light mode, editorial feel.** Clean white or warm off-white surfaces. Strong, opinionated typography that carries hierarchy. Think Linear meets a premium financial report — structured, airy, with precise accents. NOT dark-mode-with-glowing-accents (that's the AI trap this project already fell into). NOT cyan/purple gradient pallettes.

Anti-references:
- Generic AI fintech dashboards (dark, neon, glassmorphism)
- Bootstrap/Material card grids with the same padding everywhere
- "Hero number + subtitle + gradient" metric blocks

### Design Principles
1. **Typography first** — hierarchy should be readable from across a room. Size and weight do the work; color is secondary.
2. **Light with precision** — white space is not empty; it is structure. Every gap is intentional.
3. **One voice** — the interface is confident, not decorative. Remove anything that doesn't add meaning.
4. **Numbers are the UI** — financial data is the content. The chrome should disappear; the numbers should sing.
5. **Impress through restraint** — what you leave out is as important as what you put in. Hackathon judges are tired of sparkle.
